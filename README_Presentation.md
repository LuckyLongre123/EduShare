### ğŸ”— Database Schema Details

#### ğŸ‘¥ Users Table
- **Primary Key**: `user_id` (Auto-increment)

- **Security**: Password hashing with PHP `password_hash()`

#### ğŸ“š Materials Table

- **Foreign Keys**: 
  - `uploaded_by` â†’ `users.user_id` (CASCADE DELETE)
  - `college_id` â†’ `colleges.college_id` (CASCADE DELETE)
- **File Support**: PDF, DOC, DOCX, TXT, PPT, PPTX (Max 10MB)
- **Status Flow**: Pending â†’ Approved/Rejected (Admin moderation)
- **Metadata**: Title, description, subject, semester, tags

#### ğŸ« Colleges Table

#### ğŸ·ï¸ Tags Table
- **Primary Key**: `id` (Auto-increment)

#### ğŸ’– Likes Table
- **Primary Key**: `like_id` (Auto-increment)

#### ğŸ’¬ Comments Table
- **Primary Key**: `comment_id` (Auto-increment)

#### ğŸ“§ Contact Messages Table
- **Primary Key**: `message_id` (Auto-increment)

### ğŸ”— Relationships & Integrity

#### Foreign Key Constraints
- **Users â†’ Materials**: One-to-Many (User can upload multiple files)
- **Colleges â†’ Materials**: One-to-Many (College has multiple materials)  
- **Materials â†’ Likes**: One-to-Many (Material can have multiple likes)
- **Materials â†’ Comments**: One-to-Many (Material can have multiple comments)
- **Users â†’ Likes**: One-to-Many (User can like multiple materials)
- **Users â†’ Comments**: One-to-Many (User can comment on multiple materials)

#### Database Optimization
- **Indexed Fields**: Primary keys, foreign keys, email (Faster queries)

- **Data Validation**: Server-side input sanitization and type checking
